## 設計判断

- **FX-ANNOT-LIVE (#116)**: 28/28 unannotated の root cause は API 鍵不在ではなく
  **daily 経路の配線ギャップ** (`run_annotation_analytics` Step C が AI を渡さない)。
  週次 rebuild と同形で Step C にアダプタ直結。performance leakage (regime=direction_hit /
  vol=close_error_bp の循環) を OHLC 由来へ是正。決定論 OHLC fallback を `ai > auto >
  manual > ohlc_fallback` の最下位ティアとして追加 (新 pure module `annotation_fallback.py`)。
- **FX-STATE-HYSTERESIS (#117)**: same-snapshot shock damping (前日 state 不要・plumbing なし
  = brief の主アプローチ)。corroboration ゲートは `max(0, -e_star)` の負の方向強度を使用
  (`negative_e=(1-e_star)/2` は中立で 0.5 になり正予測を誤 corroboration するため)。
- **FX-MAG-EXPANSION (#118)**: conviction (=reliability, §8 Option B) と別系統のボラ拡張
  multiplier。**direction/FLAT は expansion 前の値で確定**し非 FLAT のみ符号付き magnitude を
  scale (FLAT epsilon 不変)。fire は**入力シグナル** `signal_features.fire_probability` を使用
  (ライフサイクル事後 P(fire) は負の大変動日に崩れ拡張を抑制するため)。
- **FX-STATEPROXY-REDEF (#119)**: binding AC (in-place 再定義禁止) に対し **新メトリクス追加
  パス**を選択。`state_proxy_hit` は persistence と明示 (spec)、別列 `state_correctness_hit`
  (forecast state vs OHLC 由来 realized state) を新設。24ファイル rename を回避。ORM 変更なし
  (state 診断は CSV-only)。realized 分類は LifecycleState 語彙限定・`failure` 非出力の coarse v1。
- **FX-GOV-REGIME-FLAGS (#120)**: regime/vol 層別の direction collapse フラグ。emit だけでは
  `overall_judgment` が変わらないため、flag id を **logic-flag 消費4箇所すべて**へ配線。
  collapse チェックは pooled でなく**canonical UGH variant** にアンカー (他フラグと整合)。
