## 設計判断

`engine_version` bump は出力境界が変わる箇所だけに限定した。P2B では fire 到達性と winner-margin を分離するため、fire evidence は weighted sum + catalyst floor、最終 `updated_probabilities` は独立温度 `final_softmax_temperature=0.12` の softmax 正規化にした。P3A は dynamic ratio sweep を経て、UGH の予測自体が volatility-scaled であるため default は固定 3.0bp epsilon (`ratio=0.0`, `floor=3.0`) に確定した。

## 成功パターン

PR ごとに acceptance を狭く保ち、replay sweep の数値は PR 本文に置き、脆い rate threshold は unit test に固定しなかった。review comment は原因を caller override preservation に絞って小さく修正した。

## 修正・訂正

P2B の第一レバー (`prior_weight` / `evidence_weight`) は fire と winner-margin のトレードオフで blocked と判断し、第二レバーの最終層 softmax に切り替えた。P3A の当初 acceptance にあった 5/26 type 救済は magnitude calibration の領域として外し、rare FLAT 運用に再定義した。
