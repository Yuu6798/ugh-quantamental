## 修正・訂正

- **(P1, 重要) state は forecast direction の入力ではない**。direction/FLAT は
  `projection_res.e_star`×conviction→`_direction_from_bp_with_epsilon` で決まり、
  `dominant_state` は record にコピーされるだけ (`forecasting.py:340-366`)。
  → 当初の月次分析と FX-STATE-HYSTERESIS の前提「6/12 failure→FLAT が反発を逸失」
  は**因果が誤り**。FLAT は projection/epsilon 由来で failure ラベルは並走のみ。
  ブリーフを state ラベル/state_proxy/stratification の cleanup に rescope。
- **regime/vol ラベルが performance 由来 (循環)**。`generate_ai_annotations` /
  `ai_annotations.py` は regime を `direction_hit`、vol を `close_error_bp` から
  導出 (`analytics_annotations.py:186-194`)。→「choppy 0% dir」は定義上自明で、
  月次分析の「choppy/high-vol 全敗が構造的弱点」結論は **FX-ANNOT-LIVE で市場由来
  ラベルに是正後、再検証が必要**。
- round 3 の FX-STATEPROXY 修正が FX-GOV の相互参照ドリフトを誘発 (round 4 で指摘)
  → §5 一元化の動機。
- daily アノテーションは `as_of_jst` キー、`build_labeled_observations` は
  `forecast_id` キー → 単純な CSV 受け渡しでは coverage 0 のまま (forecast_id
  変換が必須)。
